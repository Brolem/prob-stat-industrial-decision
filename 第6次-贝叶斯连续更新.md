# 第6次：贝叶斯连续更新

> 这是一份可直接照着讲的课堂讲稿。  
> 讲解时长：90 分钟。  
> 使用方式：从上到下顺序推进；遇到“请写下”“请回答”“停 1 分钟”就让学习者实际完成。  
> 本节承接第 5 次：第 5 次把过程写成分布，本节讨论新数据如何连续改变参数和可信程度。

## 开场：认识不是一次性完成

前面我们已经看到：

```text
先验 + 证据 -> 后验
```

今天要把这件事连续起来：

```text
先验 -> 第一次证据 -> 后验 -> 第二次证据 -> 新后验
```

请写下今天的课堂契约：

```text
每次新证据进入，都要说明它如何改变原判断。
```

今天的核心句是：

```text
昨天的后验，就是今天继续学习的先验。
```

此刻你应该能说清：

```text
连续更新不是反复套公式，而是把判断的时间顺序写清。
```

## 先做判断

某新设备准备投入生产。历史同类设备数据显示：

```text
设备单次试运行成功率集中在90%附近。
```

现在新设备完成 5 次试运行，全部成功。

管理者问：

```text
能否认为这台设备已经足够可靠？
```

请先写判断：

```text
第一次判断：足够可靠 / 不能直接判断 / 需要继续测试
理由：
我认为5次全成功的证据强度：
```

停 1 分钟。

现在回答：

1. 你更看重历史经验，还是新设备 5 次全成功？
2. 如果接下来又成功 5 次，你的判断会怎样变化？
3. 如果接下来失败 1 次，你的判断会怎样变化？

请写下：

```text
连续更新要同时保存旧经验和新数据。
```

此刻你应该能说清：

```text
5次全成功是证据，但它需要和历史先验合并。
```

## 第一次证据

我们用自然频数做一个简化版本。

假设对设备可靠性有两种解释：

```text
H1：设备可靠，单次成功率为95%
H2：设备一般，单次成功率为80%
```

在看到新数据之前，我们给两种解释相同权重：

```text
P(H1)=50%
P(H2)=50%
```

现在看到证据：

```text
E1 = 5次试运行全部成功
```

在两种解释下，看到这个证据的机会：

```text
P(E1|H1)=0.95^5≈77.4%
P(E1|H2)=0.80^5≈32.8%
```

请填表：

| 解释 | 先验权重 | 证据机会 | 未归一化权重 |
| --- | ---: | ---: | ---: |
| H1可靠 | 0.50 | 0.774 |  |
| H2一般 | 0.50 | 0.328 |  |

标准结果：

| 解释 | 先验权重 | 证据机会 | 未归一化权重 |
| --- | ---: | ---: | ---: |
| H1可靠 | 0.50 | 0.774 | 0.387 |
| H2一般 | 0.50 | 0.328 | 0.164 |

把两个权重归一化：

```text
总权重 = 0.387 + 0.164 = 0.551
P(H1|E1)=0.387/0.551≈70.2%
P(H2|E1)=0.164/0.551≈29.8%
```

请改写判断：

```text
第二次判断：
5次全成功把H1权重从多少提高到多少：
```

标准表达：

```text
H1权重从50%提高到约70.2%。
```

此刻你应该能说清：

```text
证据更支持可靠解释，但没有消除一般解释。
```

## 第二次证据

现在继续测试，又得到 5 次全部成功。

```text
E2 = 接下来5次也全部成功
```

这次不要回到 50%。

我们用上一步后验作为新先验：

```text
P(H1)=70.2%
P(H2)=29.8%
```

再次更新：

| 解释 | 新先验权重 | 证据机会 | 未归一化权重 |
| --- | ---: | ---: | ---: |
| H1可靠 | 0.702 | 0.774 |  |
| H2一般 | 0.298 | 0.328 |  |

标准结果：

| 解释 | 新先验权重 | 证据机会 | 未归一化权重 |
| --- | ---: | ---: | ---: |
| H1可靠 | 0.702 | 0.774 | 0.543 |
| H2一般 | 0.298 | 0.328 | 0.098 |

归一化：

```text
总权重 = 0.543 + 0.098 = 0.641
P(H1|E1,E2)=0.543/0.641≈84.7%
P(H2|E1,E2)=0.098/0.641≈15.3%
```

请写下完整更新链：

```text
H1可靠：50% -> 70.2% -> 84.7%
H2一般：50% -> 29.8% -> 15.3%
```

此刻你应该能说清：

```text
连续一致的证据，会逐步增加更能解释该证据的模型权重。
```

## 看证据强度

现在只看一次 5 连成功的证据强度。

```text
P(E|H1)=0.774
P(E|H2)=0.328
```

似然比：

```text
0.774 / 0.328 ≈ 2.36
```

解释为：

```text
5次全成功在可靠模型下出现的机会，是一般模型下的约2.36倍。
```

如果两轮证据相互独立，连续两轮的证据强度相乘：

```text
2.36 × 2.36 ≈ 5.57
```

请回答：

```text
为什么第二轮证据还会继续提高H1权重：
```

参考表达：

```text
因为同样的5连成功仍然更符合H1可靠模型。
```

此刻你应该能说清：

```text
证据强度说明新数据更偏向哪个解释。
```

## 审查独立性

连续更新最容易出错的地方是重复计算证据。

请看三种情况：

| 情况 | 能否简单连续相乘 | 原因 |
| --- | --- | --- |
| 两轮试验在不同时间、相同工况、独立记录 | 可以作为起点 | 每轮提供新证据 |
| 第二轮只是第一轮数据的重复录入 | 不可以 | 没有新信息 |
| 两轮都由同一未校准传感器判断成功 | 要审查 | 共同测量偏差会放大信心 |

请写下：

```text
连续更新需要新证据，而不是同一证据的重复包装。
```

现在做一个判断：

```text
10个传感器都报警，但10个传感器使用同一套算法。
这更像10条独立证据，还是1条共同来源证据？
```

参考表达：

```text
更接近1条共同来源证据，需要审查算法共同偏差。
```

此刻你应该能说清：

```text
贝叶斯更新可以连续进行，但证据独立性要逐次审查。
```

## 迁移到科学解释

现在换成科学问题。

有两种机理解释某材料强度下降：

```text
H1：湿度导致微裂纹扩展
H2：温度循环导致界面退化
```

新实验结果：

```text
高湿常温下强度下降明显。
低湿高温循环下强度下降较小。
```

请写：

```text
这个结果更支持哪个解释：
还不能排除哪个解释：
下一条最有价值的证据：
```

参考表达：

```text
更支持湿度导致微裂纹扩展。
还不能完全排除温度循环解释。
下一条最有价值的证据，是设计能区分湿度和温度作用的对照实验。
```

此刻你应该能说清：

```text
科学推断中的贝叶斯思维，是让证据逐步改变竞争解释的权重。
```

## 划清边界

连续更新不保证判断越来越接近真实。

它依赖：

| 依赖项 | 出错后果 |
| --- | --- |
| 先验覆盖真实解释 | 漏掉真实原因 |
| 似然描述正确 | 错估证据强度 |
| 新证据独立 | 重复计算信息 |
| 实验和测量可靠 | 系统偏差累积 |
| 模型保持适用 | 工况改变后旧模型失效 |

请写下边界句：

```text
连续更新让判断更一致，不自动保证模型更真实。
```

完成最终输出：

```text
更新链：
最强证据：
边界说明：
下一步证据：
```

此刻你应该能说清：

```text
更新结果要和证据来源、模型假设一起汇报。
```

## 连接下一课

今天我们默认了先验：

```text
H1 50%
H2 50%
```

但现实中先验不会凭空出现。

下一课要问：

```text
先验从哪里来？
模型正确和现实正确为什么不是一回事？
```

此刻你应该能说清：

```text
连续更新需要先验；先验来源和模型争议必须公开讨论。
```

## 收束：用三句话带走今天

第一句：一句更新链。

```text
在两轮5连成功后，可靠模型权重从50%更新到70.2%，再更新到84.7%。
```

第二句：一句边界说明。

```text
这个更新依赖两轮证据独立、模型似然合理、测试记录可靠。
```

第三句：一个下一课问题。

```text
第一轮更新前的先验权重，应该从哪里来？
```

## 本节板书总表

| 名称 | 直观含义 | 本节表达 |
| --- | --- | --- |
| 先验 | 新证据前的权重 | `P(H1)=50%` |
| 似然 | 解释产生证据的机会 | `P(E|H1)=0.95^5` |
| 后验 | 证据后的权重 | `P(H1|E)=70.2%` |
| 新先验 | 下一轮更新起点 | 上一步后验 |
| 似然比 | 证据偏向哪个解释 | `0.774/0.328` |
| 独立性审查 | 证据能否重复使用 | 共同来源不能重复相乘 |
| 更新边界 | 模型和证据限制 | 输入正确性、测量偏差、工况变化 |
