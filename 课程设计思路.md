# 以概率和统计为工具的科学推理与工业决策入门：课程设计总纲

## 课程定位

这门课不是概率论知识补习，也不是按知识目录平均分配课时的统计学概论。

课程面向已经完成理工科本科教育的成年人。百分比、函数、均值、方差等基础内容不单独系统重讲，而是在前两次课中通过完整问题进行诊断和即时补缺。

课程主线是：

```text
数学语言回顾 → 概率模型 → 条件化思维 → 贝叶斯逆推 → 数据如何更新认识 → 推断如何转化为决策 → 实验、因果与工业过程控制
```

课程核心不是让学习者记住更多公式，而是训练一种判断结构：

```text
先提出问题 → 再观察证据 → 再更新判断 → 最后连接行动
```

## 教学原则

1. 真实问题先行。每次课从一个科学、生活或工业问题进入，不从定义和公式进入。
2. 图形和自然频数先行。涉及条件概率和贝叶斯问题时，先用人数表、树图、二维表或图像表达，再整理成公式。
3. 公式后置。公式是对前面思考结构的压缩，不是课堂起点。
4. 证据边界明确。每次课都要说明数据支持什么、不支持什么。
5. 工业迁移贯穿。科学和生活案例之后，要回到检测、质量、设备、材料、实验或过程监控问题。
6. 核心课时倾斜。“条件概率—贝叶斯更新—科学推断”是课程思想重心，应占整个课程接近一半。

## 11 次课程主线

### 第1次：数学语言体检

- 核心问题：平均合格率更高，是否就应该扩产这条线？
- 主线推进：从生产线扩产问题进入比例、百分点、相对变化、均值、波动、图表、损失结构和数据边界。
- 必须覆盖：合格率、平均合格率、百分点、相对变化、均值与波动、短期样本与长期总体、数据支持与不支持。
- 典型案例/课堂产出：两条生产线 10 天抽检数据。学习者输出一句数据结论、一句边界说明和一个下一步问题。

### 第2次：概率模型

- 核心问题：连续 5 件产品合格，是否说明这一批产品质量很好？
- 主线推进：从短期连续合格进入事件、样本空间、频率、概率、独立性、期望和模拟。
- 必须覆盖：随机现象、样本空间、事件、频率解释、主观可信程度、独立性、期望值、短期波动。
- 典型案例/课堂产出：产品连续合格、设备短期无故障、简单抽样模拟。学习者能说明概率不是计算技巧，而是对不确定性的建模。

### 第3次：条件概率

- 核心问题：增加一条信息后，我们讨论的还是原来的总体吗？
- 主线推进：用检测、质量或分组数据说明条件改变参照总体，再进入联合概率、边缘概率和条件概率。
- 必须覆盖：`P(A|B)` 的含义、条件改变总体、二维列联表、概率树、自然频数、独立与条件独立。
- 典型案例/课堂产出：医学检测阳性与工业检测不合格。学习者能稳定区分 `P(结果|原因)` 和 `P(原因|结果)`。

### 第4次：贝叶斯第一次

- 核心问题：看到结果以后，怎样反推原因是否存在？
- 主线推进：先用人数表和树图处理医学检测与产品检测，再把结构整理成贝叶斯公式。
- 必须覆盖：先验概率、似然、后验概率、基准率、假阳性、假阴性、自然频数表示。
- 典型案例/课堂产出：罕见病检测与罕见缺陷检测。学习者能解释为什么高准确率检测不等于阳性结果高度可信。

### 第5次：随机变量与分布

- 核心问题：一个概率分布到底在描述什么？
- 主线推进：从合格/不合格、缺陷数、测量误差和寿命等待时间进入随机变量与分布。
- 必须覆盖：随机变量、参数、期望、方差、分位数、伯努利、二项、泊松、正态、寿命或等待时间模型。
- 典型案例/课堂产出：产品合格、表面缺陷数、连续质量指标、设备寿命。学习者能把分布理解为数据生成机制的模型，而不是公式清单。

### 第6次：贝叶斯连续更新

- 核心问题：新证据如何连续改变已有认识？
- 主线推进：把一次检测扩展为连续学习，连接先验、新证据、似然、后验和下一轮先验。
- 必须覆盖：连续更新、证据强度、似然比、多次证据是否独立、旧经验与新数据的权重变化。
- 典型案例/课堂产出：设备可靠性小样本测试、科学理论竞争解释。学习者能用“先验 → 证据 → 后验 → 新先验”解释连续学习。

### 第7次：先验、模型与争议

- 核心问题：先验从哪里来，模型正确是否等于现实正确？
- 主线推进：讨论先验来源、模型假设和敏感性分析，避免把贝叶斯方法神化为万能公式。
- 必须覆盖：历史数据、同类设备、物理机理、专家经验、弱信息先验、敏感性分析、模型不确定性、样本偏差。
- 典型案例/课堂产出：历史试验数据与新测试数据结合。学习者能说明先验不是随意判断，公式正确也不能保证模型描述了真实过程。

### 第8次：从推断到决策

- 核心问题：知道某件事有多可信以后，为什么仍不能直接行动？
- 主线推进：从后验概率进入行动选择，讨论损失结构如何改变决策阈值。
- 必须覆盖：期望损失、错判成本、漏判成本、决策阈值、风险偏好、信息价值、复检价值。
- 典型案例/课堂产出：产品返工、设备停机、增加检测、维修决策。学习者能区分“我相信什么”和“在损失结构下我做什么”。

### 第9次：样本支持总体

- 核心问题：样本如何支持总体结论，贝叶斯和频率学派各自回答什么？
- 主线推进：从抽样波动进入区间、检验和两种统计语言，再比较置信区间与后验可信区间。
- 必须覆盖：样本与总体、抽样波动、大数定律、中心极限定理直观、点估计、区间、假设检验、`p` 值、两类错误。
- 典型案例/课堂产出：同一份工业抽检数据分别用频率学派和贝叶斯语言表述。学习者能说明两种方法分别回答的具体问题。

### 第10次：证据不等于因果

- 核心问题：概率更新为什么不能替代实验设计和因果识别？
- 主线推进：从相关和混杂进入实验设计，说明证据来源决定结论边界。
- 必须覆盖：相关关系、因果关系、混杂变量、选择偏差、幸存者偏差、对照组、随机化、盲法、重复实验、外推。
- 典型案例/课堂产出：工艺参数改变、原材料批次变化、设备参数优化。学习者能识别“证据支持相关”与“证据支持因果”的差别。

### 第11次：工业过程综合

- 核心问题：如何判断过程是否变化，并决定继续生产、增加检测、调整参数还是停机？
- 主线推进：把概率模型、抽样推断、因果判断、损失结构和过程监控整合到一个工业问题中。
- 必须覆盖：正常波动与异常信号、历史过程基准、规格要求、报警与误报警、漏报警、行动阈值、新数据更新旧模型。
- 典型案例/课堂产出：材料关键性能指标波动，结合历史数据、在线检测、破坏性试验和设备报警作判断。学习者能定义问题、识别信息、更新判断、区分异常与因果，并说明行动条件。

## 课程阶段

### 第一阶段：诊断与共同语言

第 1-2 次负责建立共同语言。重点不是补完所有基础数学，而是通过完整问题暴露遗忘、误解和表达断点。

第 1 次已经按“判断 → 数据 → 图像 → 波动 → 再抽样 → 损失 → 迁移 → 边界 → 下一课”的结构展开。后续课程应保持这种问题驱动节奏。

### 第二阶段：条件化与贝叶斯更新

第 3-7 次是课程核心。学习者要从“会套公式”转向“知道信息如何改变总体、证据如何改变认识、模型假设如何限制结论”。

这一阶段每次课都应坚持：

```text
自然频数或图形表示 → 数学形式化 → 工业同构案例 → 改变条件后的迁移问题
```

### 第三阶段：推断、因果与决策

第 8-11 次把认识转化为行动。重点是让学习者理解：推断不是决策本身，证据不是因果本身，统计受控也不是产品必然满足要求。

这一阶段应反复要求学习者输出：

```text
我相信什么 → 这个判断的边界是什么 → 在当前损失结构下我做什么
```

## 每次课程的固定开发模板

后续逐课写讲稿时，每次课统一使用以下骨架：

1. 开场问题：先让学习者写直觉判断。
2. 证据呈现：给出表格、图像、自然频数或小型模拟。
3. 判断更新：让学习者比较前后判断变化。
4. 数学压缩：把前面结构整理为概念、公式或模型。
5. 工业迁移：换成检测、质量、设备、材料或过程问题。
6. 边界说明：写出数据支持什么、不支持什么。
7. 下一课入口：留下一个会自然引向下次课的问题。
